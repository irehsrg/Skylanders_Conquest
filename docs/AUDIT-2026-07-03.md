# Code Audit — 2026-07-03

A full audit of the C++ codebase (10 parallel reviewers + manual verification of
every finding), with fixes applied and verified via UBT (clean build, no warnings).
Everything here compiled against UE 5.6. **Nothing below requires editor work** —
open the project and playtest whenever you have time.

## Fixed (committed)

### Critical
- **Dead input after PLAY** — the menu's `FInputModeUIOnly` flags the game
  viewport as ignore-input and that survives `OpenLevel`. Starting a match from
  the main menu loaded Joust with all controls dead. Fixed in both menu handlers
  and re-asserted in `SkylandersCharacter::BeginPlay`.

### High
- **Item stats being silently deleted** — `UseMana` and `LevelUp` clamped
  current mana/health to *base* max, so every ability cast / level-up wiped
  item-granted bonus mana/HP.
- **Buff expiry corrupting stats permanently** — buffs restored absolute stat
  snapshots on expiry; leveling up (or taking a second buff) during the 90s
  window got erased forever. Removal now subtracts exactly the applied bonus.
- **Enemy god dealt half damage** — protections were applied twice (once by the
  god, once inside the player's `TakeDamage_Custom`). Also removed the duplicate
  damage numbers this produced.
- **Enemy minions spawned mis-configured** — team/type were assigned *after*
  `BeginPlay` ran, so all minions got friendly collision (player walked through
  enemy minions), blue bodies, and ranged minions never got their higher
  XP/coin rewards. Same class of bug fixed for red-side towers/phoenix/titan
  and all buff camps in `SkylandersMapBuilder` (they rendered friendly-blue).
  All of these now use deferred spawning.
- **Destroyed towers stayed as invisible walls** — they kept full collision,
  blocking movement and eating projectiles. Same for titans.
- **Tower dives unpunished during wave fights** — the tower-aggro notification
  was skipped whenever the attacked minion was busy fighting other minions.
- **Enemy base treated you as friendly** — it healed you, let you shop, and its
  fountain ignored you. Spawn areas are team-owned now: the red base fountain
  burns the player and friendly minions (backdoor protection), and the platform
  is colored per team.

### Medium
- Machine gun / Yamato movement locks were overwritten every frame by Tick's
  speed recompute (you could creep while "locked").
- Ability keys bound with `ETriggerEvent::Triggered` re-fired every frame while
  held — a held press instantly self-cancelled the machine gun toggle. Now
  `Started` (press edge).
- Ultimate rank 5 was unreachable (gate formula demanded level 21, max is 20).
  Rank-ups now unlock at 5/9/13/17/20.
- Leveling up while dead set `CurrentHealth > 0`, breaking every death-state
  check until respawn.
- Buy/sell changed `Coins` directly without the HUD "AddCoins" accumulator
  event — the Blueprint-side coin display desynced until the next passive tick.
- Dying with the shop open left an uncloseable modal over the death camera
  (`Die()` now closes shop + scoreboard).
- Jungle enemies aggro'd the player whenever *anything* damaged them, including
  your own towers shooting them.
- Buff camps never dropped aggro while their attacker was alive — they stood at
  the leash edge forever instead of returning home to heal.
- Wave scaling was off by one (wave 1 already spawned with +8%/+5%/+10%).
- 200-unit bottomless strip between the lane and the south jungle ground on the
  way to Mid Camp (fall = infinite void). South ground now meets the lane edge.
- Fountain damage ran per-frame and spawned a damage-number actor per enemy per
  frame; it now pulses at 4 Hz.

### Low / polish
- Level-3 start didn't rescale XP-to-next (level 4 cost 100 XP instead of 200).
- Player damage numbers showed pre-mitigation damage.
- Damage-number fade-out was dead code (computed scale never applied).
- IMMUNE (protected towers/titans) showed a white "0"; now shows "IMMUNE".
- First auto-attack fired from the right gun despite "start with left".
- Early machine-gun cancel skipped the end animation.
- Projectile `Lifetime`/`InitialSpeed` tuning knobs were dead (constructor-only);
  they're applied in `BeginPlay` now, so Blueprint overrides work.
- Golden Machine Gun projectiles damaged towers/titans, violating the
  auto-attack-only rule (new `bDamagesStructures` flag on the projectile).
- Mana/None buff camps were renamed/colored "Damage Buff" by `BeginPlay`;
  camps now keep their given names and get per-type colors
  (orange=damage, green=speed, blue=mana, gray=XP-only).
- `UpdateHUD` searched the widget tree ~10 times per frame during regen; the
  lookups are cached now.
- GC/lifetime hardening: `YamatoChargeVFX` and minion AI target pointers are
  `UPROPERTY()`; death/destroy/pause timer lambdas are weak-bound.
- Two C4701 (potentially uninitialized) warnings in the shop widget.

## Known issues left alone (need a design call or playtest)

- **Ranged minions can't chase**: their `AttackRange` (600) exceeds
  `AggroRange` (500), so the chase branch is dead code. Works in practice, but
  if you want ranged minions to kite/pursue, raise their `AggroRange` to ~700.
- **Enemy god's laning approach-minion branch is dead code**:
  `FindNearestEnemyMinion` caps its search at `AttackRange`, so the god never
  walks *toward* minions — it only reacts to ones already in range.
- **`MachineGunLoopAnim` is never played**: only the start anim plays. Chaining
  start → loop → end needs the actual animation lengths, so I left it.
- **Recall movement**: recall relies on Tick restoring walk speed so that
  moving cancels the channel. If you'd rather hard-lock movement during recall,
  that's a design change.

## Environment notes

- **UE 5.8 is installed, but the project still targets 5.6**
  (`EngineAssociation: "5.6"`). I deliberately did not migrate: opening/saving
  assets in 5.8 is one-way. If you want to move to 5.8, do it yourself via
  right-click the .uproject → Switch Unreal Engine Version (and expect a full
  rebuild + asset resave).
- **unreal-engine-skills-for-claude-code plugin**: to let Claude drive the
  editor via MCP, open the project in the editor and run
  `ModelContextProtocol.GenerateClientConfig ClaudeCode` in the editor console —
  that generates the `.mcp.json` at the project root. There's no `.mcp.json`
  yet, so live-editor control wasn't available for this audit.

## First-playtest checklist

1. Main menu → PLAY → confirm you can move/look/shoot immediately.
2. Enemy minions should be **red** and block your movement; red towers/titan
   red; buff camps orange/green/blue/gray by type.
3. Buy a mana item, cast an ability — max mana should stay boosted.
4. Kill the Bull Demon King, level up during the buff, wait for expiry —
   power should keep the level gains.
5. Destroy the red tower — you should be able to walk and shoot through where
   it stood.
6. Walk into the red base — the fountain should hurt (~125 per pulse), and B
   shouldn't offer buying.
7. Walk south from mid-lane toward Mid Camp — no more falling into the void.
